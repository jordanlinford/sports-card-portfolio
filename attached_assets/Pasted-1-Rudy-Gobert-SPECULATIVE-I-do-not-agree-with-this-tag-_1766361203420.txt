Gobert has none of that. His value is:

limited

boring

predictable

HOLD_CORE (most of the time)

or AVOID_NEW_MONEY if you want to be blunt about limited upside and thin demand

But never SPECULATIVE.

VETERAN/AGING stage

high volatility

then fell through to SPECULATIVE

Risk ≠ speculation.
Gobert is risky to own incrementally, but not a lottery ticket.

Why SPECULATIVE can be acceptable

Herbert is:

elite talent

but in a narrative transition phase

with market volatility tied to team results

and QB card prices that swing hard on sentiment

“High upside, high volatility — size accordingly.”

That is not insulting to Herbert.

Why SPECULATIVE can also be wrong

If SPECULATIVE is being interpreted as:

fringe

unproven

lottery-ticket-level

…then it’s too harsh.

Herbert should never be lumped with rookies or long shots.

What the verdict should be

HOLD_CORE by default

SPECULATIVE_FLYER only if:

short-term horizon

high volatility

confidence is LOW

posture text is not “lottery ticket”

If your SPECULATIVE copy says “lottery ticket only,” then Herbert should not be SPECULATIVE.

The real problem is copy, not logic (for Herbert)

You’ve overloaded SPECULATIVE with two meanings:

“Unproven long shot” (rookies, prospects)

“Volatile elite with timing risk” (Herbert, Lamar sometimes)

Those need to be visually and linguistically separated.

Two ways to fix this (pick one)
Option A (logic fix, simplest)

For franchise-core QBs:

if (
  roleTier === "FRANCHISE_CORE" &&
  verdict === "SPECULATIVE_FLYER"
) {
  verdict = "HOLD_CORE";
}


Then handle volatility via copy under HOLD_CORE:

“High volatility — don’t add aggressively.”

Option B (UX fix, better long-term)

Split SPECULATIVE into two labels:

SPECULATIVE (Emerging) – rookies / long shots

VOLATILE CORE – elite players with swingy markets

This avoids insulting elite players while preserving nuance.

Bottom line

Rudy Gobert → SPECULATIVE is wrong.
Should be HOLD_CORE or AVOID_NEW_MONEY.

Justin Herbert → SPECULATIVE depends on your copy.
If it reads “lottery ticket,” it’s wrong.
If it reads “high volatility, timing risk,” it can be okay — but HOLD_CORE is safer.
