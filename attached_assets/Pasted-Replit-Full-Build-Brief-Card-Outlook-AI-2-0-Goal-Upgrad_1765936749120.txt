Replit Full Build Brief: Card Outlook AI 2.0
Goal

Upgrade MyDisplayCase’s Buy / Watch / Sell feature so it becomes useful, honest, and trusted by collectors, with minimal user input and a dedicated single-card view. The existing Outlook currently lives inside Display Cases and requires too much manual input (player tier). We need to rebuild the feature as a true single-card intelligence system that display cases can consume.

Success Criteria

A collector can:

Click a single card and immediately see:

Market value estimate

Trend direction + chart

Liquidity indicator

Volatility indicator

Market Outlook (BUY / WATCH / SELL / LONG HOLD)

Data Confidence (High/Med/Low) with a clear reason

Short explanation they agree with (no manual tier selection)

The system auto-detects player career stage (rookie/rising/elite/legend/retired) most of the time.

User inputs required:

Only “Sports vs TCG” (and optionally time horizon 6/12/24 mo)

Everything else inferred from stored card info + market data

Rules (starter):

If “rookie” in title or set/year matches known rookie year pattern → ROOKIE

If player is retired (detected via search snippets / Wikipedia summary result) → RETIRED or LEGEND

If active and early-career (first pro year within last 1–2 seasons) → ROOKIE/RISING

If active and 5–10+ seasons → VETERAN/ELITE depending on popularity proxy (liquidity + value)

E) Signal Computation (Replace vague “AI decides” with measurable signals)

Compute these from pricePoints[]:

Liquidity score (1–10)

Based on count of sold comps:

0 comps in 365d → 1

1–2 in 365d → 2

3–5 → 4

6–10 → 6

11–20 → 8

20+ → 10
Prefer 90d, fallback to 365d with confidence penalty.

Trend score (1–10)

Compute slope or % change across time:

Use 180d as primary, 90d secondary, 365d context

Normalize: 1=strong down, 5=flat, 10=strong up

Volatility score (1–10)

Coefficient of variation (stddev/mean):

<5% → 1

5–10% → 3

10–20% → 5

20–35% → 7

35%+ → 10

Sport score (1–10)

Static mapping

Position score (1–10)

Static mapping by sport

Card type score (1–10)

Based on rarity/brand/grade

Composite scores

demandScore = weighted(liquidity, sport, position)

momentumScore = weighted(trend, -volatility)

qualityScore = weighted(cardType, careerStageAuto)

Action logic (deterministic)

We must avoid AI “vibes” here.

LITTLE_VALUE:

qualityScore low AND demandScore low AND marketValue below threshold (configurable)

LONG_HOLD:

careerStageAuto in (RETIRED, LEGEND) AND volatility low AND demand medium+

SELL:

trend high (recent run-up) AND liquidity high AND volatility rising OR momentum decelerating

BUY:

quality high AND trend low/flat (recent dip) AND liquidity medium+

WATCH:

default

AI should explain, not decide.

Data Confidence (deterministic)

HIGH: 10+ comps in last 180d AND low volatility OR strong source agreement

MEDIUM: 4–9 comps OR mixed sources OR moderate volatility

LOW: <=3 comps OR no dates OR high variance across sources
Generate confidenceReason (e.g., “12 recent sold comps in last 90 days with tight price range”).

Explanation generation (AI)

Use GPT-4o-mini only to:

Summarize why (based on computed signals + top evidence points)

Produce short explanation bullets

Produce long explanation for Pro

AI input must include:

AI output must be constrained to:
