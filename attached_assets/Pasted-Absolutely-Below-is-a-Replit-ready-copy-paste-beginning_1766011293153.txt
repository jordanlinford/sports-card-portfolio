3.2 Risk Signal Rules (deterministic)

Implement:

generateRiskSignals(profile)

Return array of risk signals.

V1 Rules (keep simple):

HIGH_PLAYER_CONCENTRATION: if top player > 15% of value

HIGH_POSITION_CONCENTRATION: if QB > 55% of value (NFL)

HIGH_ROOKIE_EXPOSURE: if rookie+rising > 45%

SMALL_MARKET_BIAS: if small market > 35%

LOW_LIQUIDITY: if lowLiquidityPct > 25%

GRADE_RISK: if raw > 40% and user is “investment” oriented (optional flag)
Each includes affected cardIds when possible (cards contributing most to signal).

System Prompt (copy/paste)
You are PortfolioOutlookAI for a sports card investing platform.
Tone: confident, honest, non-hype, collector-native. No cringe. No financial advice disclaimers beyond: “Not financial advice.”
You must be explainable. Avoid obvious takes. Use the user’s actual exposures.
Never claim certainty. Prefer “likely / tends to / historically” language.
Return ONLY valid JSON matching the schema. No markdown, no extra keys.

User Prompt Template (copy/paste)
Generate a portfolio outlook for this user based on the portfolio profile and risk signals.

Portfolio Profile:
{{PROFILE_JSON}}

Risk Signals:

Required Output JSON schema:

  "primaryDriver": "short phrase",
  "summaryShort": "1-2 sentences",

- Keep it punchy and readable. No long paragraphs.

4) Next Buys Engine (V1)
4.1 Sources for candidates (V1)

Create candidates from:

Hidden Gems results (if you already generate undervalued calls)

Watched players (pull 1–3 key card archetypes per player)

Market Outlook “BUY” cards (top confidence)

Optional: “Popular cards in Explore” if you have it (fallback only)

Store candidates in memory, then score + dedupe.

4.2 Scoring (deterministic + explainable)

Implement:

scoreNextBuyCandidate(candidate, profile)

Compute:

valueScore (0–100): discount vs comps, undervalued logic, confidence in comps

fitScore (0–100): does it reduce concentration risk? fill weak spots?

momentumScore (0–100): outlook trend, volume trend, price trend if available

overallScore = round(0.45fit + 0.35value + 0.20*momentum)

Verdict:

BUY if overallScore >= 78

WATCH if 60–77

discard < 60

4.3 “Why” bullets generator (rules-based)

Generate 3–5 bullets max:

Examples:

“Reduces QB concentration (adds prime WR exposure)”

“Priced below recent comps with solid match confidence”

“Momentum improving: WATCH → BUY shift”

“Diversifies away from single-player concentration”

“Better liquidity profile than your current average”

Also compute portfolio impact deltas:

qbExposureDelta

rookieExposureDelta

teamConcentrationDelta (if candidate team matches top team)
These can be coarse estimates using weights (assume candidate weight = candidate estPrice / portfolioValue).

4.4 Endpoint

POST /api/portfolio/next-buys/generate

Behavior:

Build profile + risk signals

Generate candidates (from above sources)

Score, dedupe by fingerprint

Save top 7 into next_buys for user with asOfDate

Return list

Caching:

If next_buys exists for today, return it unless ?refresh=true
