Behavior:

Target: eBay Sold Listings search results

Extraction Fields Per Comp

For each sold item extract:
	•	title

	•	soldDate (if shown; otherwise approximate using listing metadata)
	•	itemUrl
	•	imageUrl
	•	condition (if available)

Important:

	•	Discard non-card listings: “case”, “lot”, “pack”, “break”, “digital”, “replica”, “reprint”

Matching + Filtering (Critical)

Implement a scoring function matchScore(compTitle, filters).

Must-have filters

Hard reject if:
	•	player name not present (simple string check with common variations)
	•	year mismatch when year is provided
	•	grade mismatch when grade is required (PSA 10 vs raw)

Soft scoring

Add points for:
	•	exact set name match (“Prizm”, “Optic”, “Select”)
	•	parallel keywords (“Silver”, “Holo”, “Refractor”, “/99”)
	•	card number match if provided
	•	grading company match if graded

Subtract points for:
	•	“lot”, “break”, “pack”, “reprint”, “digital”, “case”
	•	“read description”, “see pics” (often junk)
	•	wildly different parallels (“green” when you asked “silver”)

Keep top comps by score and discard below threshold.

Confidence Score (HIGH/MED/LOW)

Confidence should reflect both quantity and match quality.

Suggested rubric:
	•	HIGH: soldCount >= 15 AND average matchScore ≥ thresholdHigh
	•	MED: soldCount 6–14
	•	LOW: soldCount <= 5 OR average matchScore below thresholdMed

Store in cache + surface to UI.
