Replit Build Ticket: Market Outlook v2 — Player = Stock, Cards = Exposure
Goal

Rebuild Market Outlook so it answers the real question collectors have:

“Should I invest in this player right now — and if yes, what card types should I buy?”

Instead of being comp-first, the experience becomes player-first:

Player = the stock

Cards = stock types / exposure vehicles (premium vs common)

Comps become secondary supporting evidence, not the product.

Success Criteria

A user can enter a player name (or arrive from a card) and see a Player Outlook.

The app outputs a clear Investment Verdict (Buy / Watch / Avoid + rationale).

The app recommends card exposure types with a “stock tier” metaphor:

Premium stock (blue-chip / low serial / autos)

Growth stock (flagship RC silvers)

Common stock (base)

Speculative (weird parallels / low liquidity brands)

The output is opinionated but transparent, avoids fake precision.

Comps/data remain visible but clearly labeled as supporting and confidence-building, not required to function.

Data + Logic Requirements
1) Player Classification Engine (Non-LLM deterministic first)

Build a lightweight rules engine that sets defaults before AI writes narrative.

Inputs (use what we can detect)

sport

position (QB premium, etc.)

career stage: Rookie / Year 2 / Prime / Aging / Retired-HOF

role status: starter / backup / unclear

recent momentum: “up / flat / down” (from available signals)

hype index: mentions/trends if available (optional)

Outputs

marketTemperature (Hot/Warm/Neutral/Cooling)

volatility (Low/Med/High)

horizon (Short/Mid/Long)

base risk profile (Low/Med/High)

Important: If we can’t detect something, set “Unknown” and let AI handle it with transparency.

2) Signals (Minimum Viable)

We do NOT need perfect data feeds to ship v1.

MVP signals you can use:

Existing comps engine output (even thin)

Google/Serper snippets you already use (news + hype)

Basic player metadata (sport/position/team) from a simple API OR internal mapping

If player metadata isn’t available:

Use AI to infer sport/position but mark as “inferred”.

3) Card Recommendation System (Deterministic framework)

Create a reusable mapping per sport that AI can fill in.

Example for Football (baseline guidance)

Premium (Blue Chip)

On-card autos

Low-numbered rookies (/99 and below)

Popular flagship parallels in liquid sets

Growth

Prizm Silver (or equivalent flagship silver)

Optic Holo / Rated Rookie Holo

Select Silver

Core

Standard rookies in strong brands

Mid-tier parallels (/199, /299)

Common

Base rookies (only for budget + volume)

Speculative

Niche sets, low collector demand brands, odd inserts

This is NOT player-specific yet. Player-specific selection happens based on position + outlook.

Caching + Refresh Strategy

Player outlook should be cache-first and updated periodically.

Cache keys

player_outlook:{sport}:{playerNameNormalized}

TTL

Hot players: refresh daily

Warm/Neutral: refresh every 3–7 days

Cooling: refresh weekly

Must support stale-while-revalidate

Return cached result immediately and refresh async.
