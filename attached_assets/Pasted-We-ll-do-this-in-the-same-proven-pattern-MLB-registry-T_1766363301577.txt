Admin UI to edit the registry instead of CSVs

This is how you stop manual file edits forever.

Design goals

Secure (admin-only)

Simple CRUD

Fast lookup (cached in-memory)

Safe migration from CSV → DB

Versioning/audit so you can undo bad edits

Architecture (recommended)

Store registry entries in Firestore (or your existing DB layer)

Load all entries into an in-memory Map on server start (and refresh every N minutes)

Admin UI updates Firestore; server cache refresh picks up changes

Decision pipeline uses “DB registry” first, then fallback to CSV (during migration), then inference

Data model

Collection: player_registry
Document id: normalized key like nfl|jamarrchase (sport + normalized name)

Fields:

sport

playerName

aliases: string[]

careerStage

roleTier

positionGroup

lastUpdated (timestamp)

updatedBy (optional)

notes (optional)

Admin UI screens

Route: /admin/registry

Features:

Search box (name/alias)

Filters: sport, roleTier, careerStage

Table view with inline edit

Create new entry modal

Bulk import CSV (paste or upload) — huge time saver

“Preview match” utility: type a name, see normalized key + what entry matches

Security

Only allow users with isAdmin: true in their user profile doc

Enforce on server/API level, not just client

Copy/paste instructions for Replit (Admin UI + Migration)

Paste this to Replit:

Build Admin Registry UI backed by Firestore, with migration from CSV

Create Firestore collection player_registry.
Document id format: ${sport.toLowerCase()}|${normalize(playerName)}
Fields: sport, playerName, aliases[], careerStage, roleTier, positionGroup, lastUpdated (server timestamp), notes (optional).

Create server endpoints:

GET /api/registry?search=&sport=&roleTier=&careerStage=&limit=

POST /api/registry (create)

PUT /api/registry/:id (update)

DELETE /api/registry/:id

POST /api/registry/import (bulk import from CSV rows)

All endpoints must require admin auth.

Add admin auth rule:

user must have isAdmin = true in Firestore users/{uid} profile OR in your existing auth claims.

If not admin: 403.

Implement client page /admin/registry:

Search input

Filter dropdowns

Data table

Add/Edit modal

Bulk import: paste CSV text area and submit; show row-level errors.

Integrate registry into analysis pipeline:

On server start, load all registry docs into an in-memory Map keyed by:

normalized playerName

normalized aliases

and also by ${sport}|${normalizedName} when sport is known

In analysis, lookup registry first.
If found: stage/roleTier/positionGroup are overridden and must not be overwritten later.
Set metaSource="registry_db".
If not found: fallback to CSV registry (metaSource="registry_csv") during migration.
If still not found: inference.

Add “migrate CSV to DB” one-time job:

Read /data/player_status_registry.csv

Upsert into Firestore player_registry

Log duplicates and alias collisions.

decisionDebug must include:
metaSource, registryId (if matched), matchedKey (name/alias), roleTier, careerStage, positionGroup.

Acceptance tests:

Kenny Pickett always AVOID.
