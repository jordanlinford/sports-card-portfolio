⭐ STEP 2 — Generate Teaser Images Server-Side (THE KEY TO VIRALITY)

Every share feature depends on the same mechanism:

🔥 A backend endpoint that returns a pre-rendered PNG image.

You will create this:

GET /api/share/case/:caseId?format=teaser
GET /api/share/case/:caseId?format=story
GET /api/share/case/:caseId?format=brag

You have 3 rendering options (choose ONE):
Option A (Best Quality): Puppeteer HTML → PNG

You already use Node, so this is very doable:

Create an HTML template that displays:

Case name

First 3–5 cards in a row

User name + tier

Total value

MyDisplayCase branding

Puppeteer loads hidden HTML and screenshots it at a custom resolution.

Pros: Perfect for TikTok, IG, Discord.
Cons: Requires Puppeteer installed (you already wanted Puppeteer for scraping).

Option B (Most Portable): Node Canvas

Draw your image manually using node-canvas.

Pros: No browser needed.
Cons: More manual work.

Option C (If you ever go Vercel): @vercel/og

This is the easiest and fastest.

⭐ STEP 5 — Discord Embed Preview (multi-card OG image)

Right now you only show one card in the preview.
We fix that by changing your Open Graph image link:

Replace your current og:image with the new generator:
<meta property="og:image" content="https://mydisplaycase.io/api/share/case/{{id}}?format=teaser" />
<meta property="og:image:width" content="1200" />
<meta property="og:image:height" content="630" />


This way:

Discord

iMessage

Twitter (X)

Facebook

…all show the full teaser image instead of a single card.

This alone will boost click-through by 2–3x.
