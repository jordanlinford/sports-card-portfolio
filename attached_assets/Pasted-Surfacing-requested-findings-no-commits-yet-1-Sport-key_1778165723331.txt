2. Current Tier-5 prompt text (line 1078–1087)
6-TIER EVIDENCE WATERFALL -- work through these in order, stop at first sufficient tier:
TIER 1 (compsAvailable>=3): BUY/SELL/HOLD/MONITOR based on comp data. SELL requires comps showing weakness.
TIER 2 (compsAvailable<3, card data known): BUY/HOLD/MONITOR. Do NOT use SELL at Tier 2.
TIER 3 (player profile data): BUY/MONITOR/AVOID/SELL if clear decline evidence.
TIER 4 (sport/position/team context only): MONITOR or AVOID.
TIER 5 (early career, unproven): Use MONITOR. Engine determines LONGSHOT_BET eligibility post-call.
TIER 6 (pre-debut ONLY): Use WATCH. Player not yet played pro games. Do NOT use WATCH for thin data.
SELL RULES: requires Tier 1 comp evidence of weakness OR Tier 3 clear decline. NOT Tier 2/4 speculation.
WATCH RULES: pre-debut only. 1+ year of pro stats = NOT WATCH. Thin data = MONITOR.

Important caveat — Gemini's response schema (line 1162) only allows "BUY|MONITOR|AVOID":

"verdict": {
  "action": "BUY|MONITOR|AVOID",
  ...
}

So Gemini physically cannot return SPECULATIVE_FLYER or SELL or LONGSHOT_BET from its primary verdict slot. The SPECULATIVE_FLYER McCarthy got must therefore be assigned post-Gemini, by the engine itself (via the INVESTMENT_VERDICT waterfall — that's the richer enum at schema.ts:1175). So the override widening to catch SPECULATIVE_FLYER is correct but it has to fire on investmentCall.verdict, not verdict.action — different field, different enum. Worth confirming this before locking the override fix.

The debug log line verdict_from_engine=AVOID on McCarthy means Tier 5 instruction was overridden by Gemini's Tier-3 AVOID (likely triggered by injury/role uncertainty — injury: RECOVERING, role: UNCERTAIN). Gemini interpreted "uncertainty" as bearish rather than "early-career upside profile."

3. Proposed strengthened Tier-5 prompt
TIER 5 — MANDATORY OVERRIDE (early-career, unproven):
For players where ALL of the following are true:
  • PLAYER DATA "Career Stage" is ROOKIE | YEAR_2 | YEAR_3 | YEAR_4 (NFL ≤4 / NBA ≤6 / MLB ≤7 / NHL ≤5 / Soccer ≤6)
  • PLAYER DATA "Position" is a skill/fantasy-relevant position
  • Player is NOT an established starter (role uncertain, backup, rotational, or injured)
  • There is no Tier 1 comp evidence of structural weakness
You MUST set verdict.action = "MONITOR".
Do NOT use BUY for these profiles (engine reserves BUY for established players with comp confirmation).
Do NOT use AVOID for these profiles. Role uncertainty, recovering injury, and unproven production
are EXPECTED for this tier — they are NOT bearish signals at this career stage. They are precisely
why the engine post-processes Tier 5 outputs into LONGSHOT_BET (small lottery-ticket position).
The engine performs LONGSHOT_BET conversion automatically based on structural eligibility
(years-in-league cutoff, skill position, role status). Your job is only to return MONITOR so the
engine can apply the conversion.
Tier 5 examples:
  • J.J. McCarthy (Year 2 QB, recovering from injury) → MONITOR
  • Spencer Rattler (Year 2 QB, backup/uncertain role) → MONITOR
  • Tyson Bagent (Year 3 QB, backup) → MONITOR
  • De'Von Achane (Year 3 RB, established starter) → NOT Tier 5; use Tier 1/3 logic (he is established)

I'd also recommend expanding the JSON schema's allowed values at line 1162 to make MONITOR an unambiguous default and remove the temptation to escalate to AVOID. Current schema "BUY|MONITOR|AVOID" is fine for Tier 5 (MONITOR is allowed); the issue is the prompt isn't forceful enough about which tier wins when signals conflict.
