. Likely Calibration Issues (Preemptively Fix These)
3.1 Momentum is probably too sensitive

Current:

momentum_score = 50 + (price_trend * 200)

Problem:

+20% move → 90 score (too aggressive)
Fix (recommended):
momentum_score = clamp(50 + (price_trend * 120), 0, 100)

This reduces overreaction.

3.2 Hype multiplier likely too strong

Current:

hype_score = 50 + (hype_raw * 150)

This can dominate composite.

Fix:
hype_score = clamp(50 + (hype_raw * 100), 0, 100)

You want hype to influence, not override.

3.3 Supply pressure slope may be too harsh

Current:

supply_score = 100 - (supply_ratio * 20)

If supply_ratio = 3 → score = 40 (very punitive)

Fix:
supply_score = clamp(100 - (supply_ratio * 15), 0, 100)

More forgiving, more realistic.

3.4 Liquidity false positives (edge case)

Problem:

2 sales / 1 listing → sell-through = 2 → looks amazing
Fix:

Add floor condition:

if sales_30d < 10:
    liquidity_score *= 0.6

This prevents fake liquidity.

3.5 Volatility over-penalization

If CV spikes slightly, score drops too fast.

Fix:
volatility_score = clamp(100 - (cv * 150), 0, 100)

Less aggressive penalty.

if price_trend < -0.05 and supply_ratio > 1.5:
