🔹 Prompt 2 – Set up Prisma + database schema

Once Prompt 1 is done, open the Replit AI/chat again and paste:

Now set up Prisma with SQLite for this project.

Initialize Prisma with npx prisma init --datasource-provider sqlite.

In prisma/schema.prisma, define three models: User, DisplayCase, and Card:

datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
}

model User {
  id                 Int           @id @default(autoincrement())
  email              String        @unique
  passwordHash       String
  stripeCustomerId   String?
  subscriptionStatus String        @default("FREE") // FREE or PRO
  displayCases       DisplayCase[]
  createdAt          DateTime      @default(now())
}

model DisplayCase {
  id           Int      @id @default(autoincrement())
  user         User     @relation(fields: [userId], references: [id])
  userId       Int
  name         String
  description  String?
  isPublic     Boolean  @default(true)
  cards        Card[]
  createdAt    DateTime @default(now())
}

model Card {
  id             Int          @id @default(autoincrement())
  displayCase    DisplayCase  @relation(fields: [displayCaseId], references: [id])
  displayCaseId  Int
  title          String
  imagePath      String
  set            String?
  year           Int?
  grade          String?
  purchasePrice  Float?
  estimatedValue Float?
  sortOrder      Int          @default(0)
}


Run the migration (npx prisma migrate dev --name init) and generate the client (npx prisma generate).

In index.js, import PrismaClient from @prisma/client, create a prisma instance, and make sure it’s available for routes.

🔹 Prompt 3 – Add auth (signup / login / logout)

After Prisma is wired up, ask Replit:

Now add basic authentication with email + password.

Install bcryptjs if it’s not installed.

In index.js, import bcryptjs.

Create routes and views for:

GET /signup → renders signup.ejs with a form for email + password.

POST /signup → hashes the password with bcrypt, creates a User via Prisma, stores { id, email, subscriptionStatus } in req.session.user, then redirects to /dashboard. If the email is taken, re-render signup with an error.

GET /login → renders login.ejs with a form for email + password.

POST /login → checks the user, compares password with bcrypt, on success sets req.session.user and redirects to /dashboard; on failure re-render with an error.

POST /logout → destroys the session and redirects to /.

Create a requireAuth middleware that checks req.session.user and redirects to /login if not logged in.

Update layout.ejs so it shows:

“Login / Signup” links when no user is logged in

“Dashboard / Logout” when a user is logged in.

🔹 Prompt 8 – Stripe checkout for upgrading to Pro

Finally, wire payments.

Integrate Stripe Checkout for upgrading to Pro.

Make sure stripe is imported and configured in index.js using process.env.STRIPE_SECRET_KEY.

Use an env variable STRIPE_PRICE_ID for the monthly subscription price.

Add a POST /create-checkout-session (requireAuth) route that:

Creates a Stripe Checkout Session in subscription mode.

Uses the logged-in user’s email as customer_email.

Uses STRIPE_PRICE_ID for the line item.

Sets:

success_url to ${process.env.BASE_URL}/billing/success?session_id={CHECKOUT_SESSION_ID}

Redirects to session.url.

In upgrade.ejs, add a form with method POST to /create-checkout-session and a submit button “Upgrade with Stripe”.

Add GET /billing/success (requireAuth) route that:

Uses the session_id from the query string.

Fetches the Checkout Session from Stripe.

Verifies the customer_details.email matches the logged-in user’s email.

Updates the User’s subscriptionStatus to PRO and sets stripeCustomerId.

Updates req.session.user.subscriptionStatus to PRO.

Renders billing_success.ejs saying “You are now Pro! Unlimited display cases unlocked.”

You can skip webhooks for now; use this success handler as the MVP.
