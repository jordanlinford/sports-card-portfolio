Implement explicit override: Victor Wembanyama must be SPECULATIVE_FLYER

Context
We have a canonical test set where all cases pass except Victor Wembanyama, who is returning ACCUMULATE. For product trust, we want Wembanyama to default to SPECULATIVE_FLYER (high upside, high uncertainty) regardless of valuation signals for now.

Requirements

Add a deterministic override so Victor Wembanyama always returns verdict = "SPECULATIVE_FLYER".

Confidence must be LOW when compSource is "modeled" (existing cap still applies).

Add decisionDebug fields so we can confirm the override triggered.

Make it easy to remove later (single map entry or config list).

Implementation
A) Create a small override config in the decision engine layer (or a shared config file):

const VERDICT_OVERRIDES: Record<string, InvestmentVerdict> = {
"victor wembanyama": "SPECULATIVE_FLYER",
"wembanyama": "SPECULATIVE_FLYER"
};

Use your existing player identifier if you have playerId. Prefer playerId if available.

B) In the verdict decision function, before returning any computed verdict (after stage classification is fine, but before final return), do:

Normalize the player name (lowercase, trim, remove punctuation).

If the normalized name matches a key in VERDICT_OVERRIDES, replace verdict with the override and set verdictReason = "Override: canonical test expectation" and decisionDebug.overrideApplied = true.

C) Do not allow ACCUMULATE for this player after override.

D) Ensure the UI still shows normal sections but the verdict is SPECULATIVE_FLYER and posture copy matches speculative.

Acceptance Criteria

Victor Wembanyama always returns SPECULATIVE_FLYER.

No other player verdict changes due to this change.

decisionDebug shows overrideApplied=true and overrideKey used.

Confidence remains capped correctly when comps are modeled.

Notes
This is an intentional product decision for now. Later we can remove the hardcode and re-allow ACCUMULATE for emerging players only under stricter thresholds.
